# 01. 产品交互设计

## 1. 产品定位

本产品不是传统 AI 写作 App，也不是 Codex Desktop 的换壳版本。它是一个 **Manuscript Runtime Client**：

- 普通用户看到的是文稿阅读、批注、AI 修改建议、规则中心、版本历史。
- 背后由 Runtime 管理所有文稿状态、注释、规则、patch、Git。
- 需要 AI 执行时，Runtime 调用 Codex app-server，并把任务限定在当前项目和当前 skill scope 内。

一句话：

> 读 Markdown、选中一段、加注释、点 AI 处理、看 diff、接受并提交。

## 2. 主交互闭环

### 2.1 打开项目

用户启动 App 后看到项目首页：

- 最近项目列表。
- 每个项目显示书名、类型、章节数、待处理批注数、未审章节数、最近提交时间。
- 进入项目后默认打开上次阅读章节。

### 2.2 读稿与批注

主界面是 Markdown 阅读/编辑器。

用户操作：

1. 打开 `chapters/ch05.md`。
2. 读到一段不满意。
3. 选中文字。
4. 右键菜单：`添加注释`、`AI 修改此处`、`AI 扩写`、`AI 缩写`、`AI 改语气`、`提取为规则`。
5. 选择 `添加注释`。
6. 输入批注，例如：
   - “这里太像 AI 了，不要解释内心，用动作表现。”
   - “这段节奏太快，建议拉长动作描写。”
   - “这和第 3 章设定冲突。”
7. 批注保存到 `.bookai/annotations.jsonl`，正文只保留或自动生成 block anchor。

### 2.3 AI 处理

用户可以在三种范围启动 AI：

- 当前选区。
- 当前章节。
- 当前项目中的所有未审章节。

启动后弹出 AI 处理面板：

- 选择 Skill：`revise-with-annotations`、`extract-writing-rules`、`propagate-rules`。
- 选择范围：选区 / 本章 / 未审章节 / 自定义范围。
- 选择模型：默认 Codex app-server，可路由到 Gemini/OpenAI/Claude，但文件操作仍由 Runtime 管理。
- 展示安全提示：不会直接写入正文，所有修改将先生成 patch。

### 2.4 AI 运行中

运行面板以事件流展示：

- 已读取当前章节。
- 已读取注释 5 条。
- 已读取规则 18 条。
- 已排除 locked 章节 3 个。
- 正在调用 skill：`revise-with-annotations`。
- 正在生成 patch。
- 需要用户确认：是否把 AN-041 提炼为全局规则？

用户可以中途取消。

### 2.5 Diff 审核

AI 处理完成后，不直接改文稿，而是进入 Diff 审核页：

左侧：原文。

右侧：建议修改。

右栏：来源批注、引用规则、置信度、影响范围、Git 提交预览。

用户可选择：

- 拒绝。
- 局部接受。
- 接受当前 patch。
- 接受并生成 Git commit。
- 接受但不提交，保持工作区脏状态。

默认建议：接受后立即 commit。

### 2.6 规则沉淀

如果多条批注反复表达同一种审美判断，系统建议生成规则：

```yaml
id: R-018
type: style
text: 人物心理优先通过动作、停顿、回避和场景压力体现，避免直接解释。
source_annotations:
  - AN-041
  - AN-044
apply_to:
  - draft
  - unreviewed
exclude:
  - reviewed
  - locked
status: active
```

用户确认后规则进入 `rules.yaml`，并可选择是否传播到后续未审章节。

## 3. 页面列表

### 3.1 项目首页

核心元素：

- 项目卡片。
- 当前项目健康度。
- 待处理批注。
- 最近 Git 提交。
- 快捷入口：新建项目、导入文稿、打开上次章节。

### 3.2 文稿编辑页

核心元素：

- 左侧章节目录。
- 中间 Markdown 阅读/编辑器。
- 右侧批注面板。
- 顶部 AI 操作按钮。
- 底部状态栏：字数、选区、章节状态、保存状态。

### 3.3 AI 运行页

核心元素：

- 当前 Skill。
- 当前 scope。
- 已读取上下文。
- Codex app-server stream 事件。
- 用户输入/确认卡片。
- 取消按钮。

### 3.4 Diff 审核页

核心元素：

- 原文/修改后双栏。
- 来源批注。
- 应用规则。
- 影响范围。
- 安全检查结果。
- 拒绝 / 局部接受 / 接受并提交。

### 3.5 规则中心

核心元素：

- 全部规则列表。
- 按风格、结构、设定、节奏分类。
- 每条规则的来源批注。
- 生效范围。
- 启用/禁用。
- 传播到未审章节。

### 3.6 Skill 中心

核心元素：

- 内置 skills。
- Codex scope skills。
- 自定义 skills。
- skill 详情。
- 最近运行记录。
- 权限需求。

### 3.7 设置 / 安全中心

核心元素：

- AI 不直接写入文稿：开关默认开启且不可关闭，除非进入高级模式。
- 修改 reviewed 章节需二次确认。
- locked 章节禁止修改。
- 所有修改生成 patch。
- 应用后自动 Git commit。
- app-server 连接状态。
- Skill scope 配置。
